Administrator Guide

System Overview

Welcome to the SAAIO Training Grounds Management Console. This platform facilitates student onboarding, progress monitoring, and cohort management across the SAAIO, IDC, and WRP programs. This guide outlines the core operational workflows required for efficient administration.

The Admin Dashboard

The primary interface provides a consolidated view of student activity. Administrators can navigate between program-specific datasets using top-level tabs. The dashboard enables granular control over individual student records, including password resets and record updates.

Section; Administrator Guide, focuses on dashboard navigation, student management, and the automated onboarding pipeline.

Automated Onboarding Pipeline

The onboarding pipeline is a cloud-integrated system that retrieves student application data from primary registration sources. It is designed to minimize manual data entry and ensure data integrity.

1
Access the Pipeline: Navigate to the Onboarding tab to view pending applications. This view groups applicants by Campus and Program for structured processing.
2
Manage Freshness: Utilize the data filters to identify new applicants. The system automatically cross-references identity records to flag existing members, preventing secondary account creation.
3
Batch Activation: Select the desired groups and define a Cohort Identity. Upon activation, the system generates secure credentials and distributes them via the integrated notification service.
Data Protection: The activation process includes an automated "Safety Lock" that overrides any accidental selection of existing students, ensuring that production records remain unique and consistent.

Cohort Management

Effective training requires organized student grouping. The platform provides tools for cohort creation and lifecycle management.

Creation: Define new cohorts with specific start dates and geographical locations to maintain accurate training schedules.
Organization: Reassign students between cohorts to accommodate scheduling changes or program transfers.

Certification Workflows

Student success is monitored through objective assessments. Once a learner reaches the completion threshold, their status is flagged for final review.

Administrators should verify exam scores and module completion percentages before final certification distribution. Verified students will have their graduation status updated across the centralized database.
