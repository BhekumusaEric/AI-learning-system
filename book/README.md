## Content Format

### Read Pages
- Markdown format with explanations
- Key concepts in bullet points
- Code examples
- "Remember" boxes with important takeaways

### Practice Pages
- Theory content (if needed)
- Initial code stub for students
- Hidden test descriptions
- Helpful hints for common mistakes

## Remaining Content

The remaining pages follow the same format and can be generated following these patterns:

1. **Read Pages**: Simplified explanations of concepts with examples
2. **Practice Pages**: Code challenges with initial stubs and hidden unit tests
3. **Progressive Difficulty**: Each chapter builds on previous knowledge
4. **IOAI Focus**: Content aligned with International Olympiad in Artificial Intelligence requirements

## Implementation Notes

- All practice pages include hidden tests that run automatically
- Feedback provides clear pass/fail with helpful hints
- Code editors support syntax highlighting and autocomplete
- Platform tracks progress and awards XP for completion

This structure provides a complete foundation for the IOAI Training Grounds learning platform.

## Solutions & Runner

- **Solutions folder:** A parallel `solutions/` folder contains solved versions of all practice lessons (each file has a `## Solution` section with working code).
- **Course runner:** Run `python3 course_runner.py --list` to list lessons.
  - Use `python3 course_runner.py --run-all` to execute hidden tests (it will prefer solution code when available).
  - Use `python3 course_runner.py --student --run-all` to run **student mode**, which executes only the practice blocks and reports which lessons still fail.
